# Security Policy

## Reporting A Vulnerability

Do not open public GitHub issues for security reports.

Use GitHub private vulnerability reporting for this repository:
- Security tab -> Advisories -> Report a vulnerability
- Direct link: https://github.com/jarrydaubert/prosepal/security/advisories/new

Include:
- Affected version/branch/commit
- Reproduction steps or proof of concept
- Impact assessment
- Any known workaround

## Response Targets

- Initial acknowledgement: within 3 business days
- Triage decision: within 7 business days
- Status updates: at least weekly until resolved

## Disclosure Process

- Reports are triaged privately.
- Fixes are prepared in private where possible.
- Public disclosure happens after a fix is available or mitigations are in place.

## Scope

This policy covers the mobile app repository and related release/security workflows.

For implementation-level controls and architecture details, see:
- `docs/SECURITY.md`
